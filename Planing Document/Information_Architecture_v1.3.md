# DSM (Daily Schedule Managements) 정보 구조도 (IA) - v1.3

> **v1.2 → v1.3 주요 변경 사항**
> - 백엔드 프레임워크 변경(NestJS) 및 실시간 WebSocket 통신 복구
> - 온보딩에 어뷰징 상한 정책 안내 추가 (0.5.2)
> - WebSocket 기반의 실시간 랭킹 변동 알림/UI 업데이트 흐름 명시

## 1. 문서 개요
본 문서는 DSM 애플리케이션의 화면 계층 구조와 사용자의 이동 흐름을 정의합니다. 진입 단계인 '인증·온보딩 흐름'과, 로그인 완료 후 하단 탭 바(Bottom Tab Bar)를 통해 이동하는 '메인 흐름'으로 구분됩니다.

---

## 2. 화면 계층 구조 (Menu Hierarchy)

### [Root] 인증 및 진입 흐름 (Auth & Onboarding Flow)

앱 실행 시 계정 연동 여부를 확인하고 진입하는 화면입니다.

* **0.1 스플래시 화면 (Splash)**
  * 앱 로고 노출 및 자동 로그인(JWT 토큰 검증) 여부 체크
* **0.2 로그인 화면 (Login)**
  * 소셜 로그인 버튼 (Google, Kakao, Apple)
* **0.3 회원가입 및 프로필 설정 (초기 1회)**
  * 닉네임 입력 및 중복 확인
  * 프로필 이미지 업로드 (선택)
  * 서비스 이용약관 및 개인정보 처리방침 동의
* **0.4 알림 권한 요청 화면**
  * FCM 푸시 권한 요청에 대한 사전 설명 (왜 필요한지, 어떤 알림을 받는지)
  * OS 시스템 권한 다이얼로그 호출 (권한 허용 시 기기의 FCM 토큰을 추출하여 서버에 전송)
  * 거부 시에도 인앱 알림은 동작함을 안내
* **0.5 온보딩 / 튜토리얼 (초기 1회)**
  * 0.5.1 일과 등록 방법 안내
  * 0.5.2 점수 산정 방식 및 **어뷰징 상한(일일 획득 가능 한도)** 안내
  * 0.5.3 랭킹·티어 시스템 안내
  * 건너뛰기 가능

---

### [Main Tab] 메인 서비스 흐름 (Tab Navigation)

로그인이 완료된 사용자가 하단 탭을 터치하여 각 페이지로 자유롭게 이동할 수 있는 구조입니다.

#### 1.0 홈 (Home) - Tab 1

사용자의 당일 일정 관리와 달성도를 직관적으로 확인하는 화면입니다.

* **1.1 메인 대시보드**
  * 1.1.1 오늘의 요약 (당일 점수, 현재 티어 뱃지, 상위 백분위)
  * 1.1.2 주간 캘린더 뷰 (날짜별 달성률 인디케이터 포함)
  * 1.1.3 당일 타임라인 기반 일과 리스트 (체크박스, 시간, 카테고리, 난이도 표시)
* **1.2 일과 관리 팝업/화면**
  * 1.2.1 신규 일과 생성 (내용, 시작/종료 시간(UTC 자동 변환), 카테고리, 난이도 입력)
  * 1.2.2 기존 일과 수정 및 삭제 기능
  * 1.2.3 알림 ON/OFF (개별 일과 단위)
* **1.3 월간 캘린더 뷰**
  * 1.3.1 전체 달력 및 일별 달성 현황 요약
  * 1.3.2 날짜 탭 시 해당일 상세 리스트 표시
* **1.4 카테고리 관리**
  * 1.4.1 기본 카테고리 (공부/운동/업무/기타) 조회
  * 1.4.2 사용자 정의 카테고리 추가/수정/삭제
  * 1.4.3 카테고리별 색상 지정

#### 2.0 랭킹 (Ranking) - Tab 2

게이미피케이션 요소를 통해 전체 사용자 간의 경쟁 및 본인의 성취도를 확인하는 화면입니다. (WebSocket 연결을 통한 실시간 데이터 수신)

* **2.1 전체 랭킹 보드**
  * 2.1.1 내 순위 하이라이트 패널 (티어 뱃지 + "현재 상위 X% 입니다") - **실시간 순위 변동 애니메이션** 적용
  * 2.1.2 랭킹 단위 선택 탭 (일간 / 주간 / 누적)
  * 2.1.3 TOP 100 랭킹 리스트 (티어 뱃지, 프로필 썸네일, 닉네임, 점수)
* **2.2 나의 통계 상세**
  * 2.2.1 최근 7일 점수 획득 추이 그래프
  * 2.2.2 누적 일과 달성률 및 카테고리별 통계 (도넛 차트)
  * 2.2.3 티어 승급 진행도 (다음 티어까지 남은 점수)

#### 3.0 마이페이지 (My Page) - Tab 3

사용자의 개인 계정 정보 및 애플리케이션 환경을 설정하는 화면입니다.

* **3.0 프로필 헤더**
  * 프로필 이미지, 닉네임, 현재 티어 뱃지 노출
* **3.1 계정 관리**
  * 3.1.1 프로필 정보 (닉네임 + 프로필 이미지) 조회 및 수정
  * 3.1.2 소셜 로그인 연동 상태 확인
  * 3.1.3 로그아웃 및 회원 탈퇴 (로그아웃 시 DB의 FCM 토큰 무효화)
* **3.2 앱 설정**
  * 3.2.1 전체 알림 ON/OFF 토글
  * 3.2.2 알림 방식 설정 (소리, 진동, 무음 등)
  * 3.2.3 OS 알림 권한 상태 확인 및 설정 앱 이동
* **3.3 고객 지원 및 정보**
  * 3.3.1 공지사항 및 도움말
  * 3.3.2 점수 산출 방식 및 랭킹/티어 규칙 안내
  * 3.3.3 앱 버전 정보
  * 3.3.4 약관 및 개인정보 처리방침 조회

## 3. 주요 사용자 플로우 (Critical Paths)

### 3.1 첫 진입 → 첫 일과 등록까지
`0.1 Splash → 0.2 Login → 0.3 프로필 설정 → 0.4 알림 권한(FCM 토큰 발급) → 0.5 온보딩 → 1.1 홈 → 1.2.1 신규 일과 생성`

### 3.2 정각 알림 수신 → 일과 완료 체크
`(백그라운드) OS 푸시 수신 → 앱 진입 → 1.1.3 당일 리스트에서 해당 일과 체크 → (API 호출) → 점수 갱신`

### 3.3 랭킹 확인 (WebSocket 실시간 업데이트)
`(하단 탭) 2.0 랭킹 진입 → WebSocket 커넥션 수립 → 2.1.1 내 순위 실시간 구독 → 다른 유저의 점수 변동 시 내 화면에 즉시 반영`
