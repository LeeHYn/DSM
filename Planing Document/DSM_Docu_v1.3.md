# 일과 관리 모바일 애플리케이션 요구사항 정의서 (v1.3)

> **v1.2 → v1.3 주요 변경 사항**
> - 백엔드 아키텍처를 **NestJS(Node.js) + 자체 JWT 인증**으로 전면 개편
> - 스케줄러를 외부 서비스(QStash)에서 **내부 node-cron**으로 변경
> - **WebSocket**을 통한 실시간 데이터 동기화 일부 복구
> - 어뷰징 방지(일일 획득 점수 상한 및 등록 개수 제한) 정책 추가
> - 타임존 처리(UTC 통일) 및 오프라인 동기화 충돌 방지 정책 명시

## 1. 프로젝트 개요
* **프로젝트명:** DSM (Daily Schedule Managements)
* **목적:** 사용자가 자신의 일과를 계획 및 실행하고, 점수화된 데이터를 통해 전체 사용자 중 자신의 성실도를 파악하며 동기부여를 얻는 일과 관리 애플리케이션.
* **주요 가치 제안:**
  * 체계적인 할 일 및 일정 관리
  * 달성률 기반의 점수제와 전체 유저 랭킹/티어 시스템을 통한 강력한 게이미피케이션(Gamification) 경험
  * **FCM 푸시 + 인앱 로컬 타이머 + WebSocket**을 결합한 신뢰성 높은 실시간/정각 알림 시스템

---

## 2. 타겟 유저 (Target Audience)
* **메인 타겟:**
  * 매일의 일과를 게임 퀘스트처럼 즐겁게 달성하고 싶은 사람
  * 타인과의 비교(랭킹/상위 %)를 통해 긍정적인 자극과 동기부여를 받는 사용자
* **유저 페르소나의 핵심 니즈:**
  * "내가 얼마나 갓생(부지런한 삶)을 살고 있는지 객관적인 수치와 랭킹으로 보고 싶다."
  * "일정 시간이 되면 절대 놓치지 않도록 정확하고 즉각적인 알람이 울리면 좋겠다."

## 3. 핵심 기능 요구사항 (Functional Requirements)

| 구분 | 기능명 | 상세 요구사항 | 우선순위 |
| :--- | :--- | :--- | :--- |
| **작업 관리** | 할 일(To-do) 및 일정 관리 | - 할 일 추가/수정/삭제 및 시작/종료 시간 설정 (서버 UTC 기준 통일)<br>- 카테고리(공부/운동/업무 등) 및 난이도(하/중/상) 선택<br>- 캘린더 뷰를 통한 월간/주간 스케줄 확인 | **High** |
| **게이미피케이션** | 성실도 점수 산정 및 어뷰징 방지 | - **점수 공식:** `Σ(완료한 일과의 난이도 점수) × 달성률 보정 계수`<br>- **난이도 점수:** 하(10점) / 중(20점) / 상(30점)<br>- **달성률 보정:** 100%→×1.5, 80% 이상→×1.3, 60% 이상→×1.0, 60% 미만→×0.7<br>- **[어뷰징 제한]**: 1일 최대 등록 가능 일과 수는 20개로 제한하며, 하루 최대 획득 가능 점수는 상한선(Cap)을 둠 | **High** |
| **게이미피케이션** | 랭킹 및 백분위 조회 | - 일간/주간/누적 랭킹 제공<br>- 전체 유저 데이터 기반 내 순위 및 상위 % 표시<br>- WebSocket을 통해 본인 및 상위권 유저의 점수 변동 시 실시간 랭킹 갱신 | **High** |
| **게이미피케이션** | 티어 시스템 | - 누적 점수 기준 6단계 티어 (브론즈→실버→골드→플래티넘→다이아몬드→마스터)<br>- 티어 승급/강등 시 인앱 알림 및 마이페이지 표시 | Medium |
| **알림 시스템** | 하이브리드 정각 알림 및 토큰 관리 | - **FCM 토큰 관리:** 유저 로그인 및 앱 실행 시 기기의 고유 FCM 토큰을 DB에 저장 및 갱신<br>- **백그라운드/종료 상태:** NestJS 내부 `node-cron`을 통한 정각 FCM(OS 푸시) 발송<br>- **포그라운드(인앱) 상태:** WebSocket 실시간 메시지 + 로컬 타이머 결합으로 즉각 반응 | **High** |
| **회원 관리** | 자체 JWT 유저 계정 연동 | - 소셜 로그인(구글, 카카오, 애플) 및 유저 프로필(닉네임, 프로필 이미지) 관리<br>- NestJS 백엔드에서 자체 발급하는 JWT 기반 세션/인증 처리 | Medium |

## 4. 비기능 요구사항 (Non-Functional Requirements)

* **알림 신뢰성 및 타임존 통일 (중요):**
  * 일과 시작/종료 시각은 클라이언트가 서버로 전송할 때 반드시 **UTC 기준 타임스탬프**로 변환하여 DB에 저장합니다.
  * FCM 토큰을 서버(PostgreSQL)에 안전하게 저장하여 푸시 유실을 방지합니다.
* **오프라인 데이터 동기화 충돌 해결:**
  * 일과 CRUD는 오프라인 시 로컬(AsyncStorage/SQLite)에 임시 저장됩니다.
  * 네트워크 복구 시 서버와 동기화하며, 데이터 충돌 시 레코드의 `updated_at` (가장 최근 수정 시각)을 비교하여 **최신 데이터를 우선 반영(Last-Write-Wins)** 합니다.
* **성능 및 데이터 처리:**
  * 랭킹 데이터는 Redis 기반으로 캐싱되며, WebSocket을 통해 변동분만 실시간으로 브로드캐스트합니다.
* **플랫폼:** iOS 및 Android (React Native 확정)
